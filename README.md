## 📌 Overview

**TaskFlow** is an enterprise-grade task management platform that allows teams to organize projects, assign tasks, track progress, and collaborate through comments — all under a clean, layered architecture.

Built as a learning + portfolio project to demonstrate real-world enterprise patterns using **ABP Framework**, **Clean Architecture**, and **Angular** frontend integration.

---

## ✨ Features

| Feature | Description |
|--------|-------------|
| 🗂️ **Projects** | Create and manage projects with status tracking |
| ✅ **Tasks** | Assign tasks with priority, due dates, and status |
| 💬 **Comments** | Collaborate on tasks with threaded comments |
| 🔐 **Authentication** | OpenIddict-based OAuth2 with JWT |
| 👥 **Role-based Access** | Admin sees all, Users see their own tasks |
| 📋 **Audit Logging** | Every change on a task is automatically logged |
| ⏰ **Background Jobs** | Reminders before task due dates |
| 🌍 **Multi-tenancy** | Ready to support multiple companies |
| 🌐 **Localization** | Arabic & English support |
| 📡 **Domain Events** | Triggers when a task is marked as Done |

## 🛠️ Tech Stack

### Backend
- **ASP.NET Core 10** — REST API
- **ABP Framework 10** — Enterprise application framework
- **Entity Framework Core** — ORM & migrations
- **SQL Server** — Database
- **AutoMapper** — Object-to-object mapping
- **OpenIddict** — Authentication & authorization server

### Frontend
- **Angular** — SPA framework
- **ABP Angular Packages** — UI theme, localization, auth
- **Bootstrap** — Responsive UI components

### Architecture
- **Clean / Layered Architecture** (Domain → Application → Infrastructure → API → UI)
- **Repository Pattern** — via ABP's generic repositories
- **CQRS-inspired** AppServices with DTOs

## 🚀 Getting Started

### Prerequisites

- [.NET 10 SDK](https://dotnet.microsoft.com/download)
- [Node.js LTS](https://nodejs.org/)
- [ABP CLI](https://abp.io/docs/latest/cli) — `dotnet tool install -g Volo.Abp.Cli`
- [SQL Server](https://www.microsoft.com/en-us/sql-server)
- [Angular CLI](https://angular.io/cli) — `npm install -g @angular/cli`

---

### 1️⃣ Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/TaskFlow.git
cd TaskFlow
```

---

### 2️⃣ Configure the database

Open `aspnet-core/src/TaskFlow.HttpApi.Host/appsettings.json` and update the connection string:

```json
"ConnectionStrings": {
  "Default": "Server=YOUR_SERVER;Database=TaskFlow;Trusted_Connection=True;TrustServerCertificate=True"
}
```

Do the same in `aspnet-core/src/TaskFlow.DbMigrator/appsettings.json`:

```json
"ConnectionStrings": {
  "Default": "Server=YOUR_SERVER;Database=TaskFlow;Trusted_Connection=True;TrustServerCertificate=True"
},
"OpenIddict": {
  "Applications": {
    "TaskFlow_App": {
      "ClientId": "TaskFlow_App",
      "RootUrl": "http://localhost:4200"
    }
  }
}
```

---

### 3️⃣ Run database migrations

```bash
cd aspnet-core/src/TaskFlow.DbMigrator
dotnet run
```

Wait for: `Successfully completed all database migrations.`

---

### 4️⃣ Run the backend

```bash
cd aspnet-core/src/TaskFlow.HttpApi.Host
dotnet run
```

API will be available at: `https://localhost:44332`  
Swagger UI: `https://localhost:44332/swagger`

---

### 5️⃣ Run the Angular frontend

```bash
cd angular
npm install
ng serve --port 4200 -o
```

App will open at: `http://localhost:4200`

**Default credentials:**
- Username: `admin`
- Password: `1q2w3E*`

## 📐 Architecture Overview

```
Angular (Frontend)
      │
      │  HTTP / REST
      ▼
HttpApi.Host  ←── ASP.NET Core + ABP Auto Controllers
      │
      ▼
Application Layer  ←── AppServices + DTOs + AutoMapper + Permissions
      │
      ▼
Domain Layer  ←── Entities + Enums + Domain Events + Business Rules
      │
      ▼
EntityFrameworkCore  ←── DbContext + Migrations + Configurations
      │
      ▼
SQL Server Database
```

---

## 🔄 Typical Backend Flow

```
Angular Form
    ↓ HTTP POST
TaskAppService.CreateAsync(dto)
    ↓ Validate + Map
Repository.InsertAsync(entity)
    ↓ EF Core
SQL Server (saved)
    ↓ Map back
TaskDto returned to Angular
```
